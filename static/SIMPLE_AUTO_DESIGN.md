# 8BIT Trading Bot - 간단한 완전 자동화 설계

## 🎯 핵심 아이디어

**사용자는 옵션만 설정하고, 나머지는 모두 자동으로 동작**

---

## 📋 사용자가 설정하는 옵션 (최소한)

### 1. 기본 설정 (한 번만 설정)

```
✅ 자동화 모드: ON/OFF
✅ Paper 모드: ON/OFF (실거래 여부)
✅ 최대 일일 손실: % (기본값: 5%)
✅ 거래 금액: KRW (기본값: 5,000원)
```

### 2. 고급 옵션 (선택적, 기본값 사용 가능)

```
✅ 리스크 레벨: 낮음/보통/높음 (기본값: 보통)
✅ 거래 빈도: 낮음/보통/높음 (기본값: 보통)
```

**끝!** 이게 전부입니다.

---

## 🤖 자동으로 처리되는 것들

### 완전 자동화 항목

1. **봇 시작/중지** → 자동
2. **분봉 선택** → 자동 (최적 분봉 자동 선택)
3. **N/B 파라미터** → 자동 최적화
4. **EMA 설정** → 자동 최적화
5. **ML 모델 학습** → 자동 (30분마다)
6. **백테스트** → 자동 (5분마다)
7. **거래 결정** → 자동 (신호 기반)
8. **포지션 관리** → 자동
9. **손절/익절** → 자동
10. **파라미터 튜닝** → 자동 (1시간마다)

## ⚙️ 자동화 로직 (간단 버전)

### 1. 시작 시 자동 실행

```
시스템 시작
  ↓
자동화 모드 확인
  ↓
IF 자동화 ON:
  → 봇 자동 시작
  → 최적 분봉 자동 선택
  → 파라미터 자동 로드
  → 거래 루프 시작
```

### 2. 주기적 자동 작업

```
매 5초:  거래 신호 확인 및 실행
매 5분:  백테스트 실행
매 30분: ML 모델 재학습
매 1시간: 파라미터 자동 최적화
```

### 3. 자동 의사결정

```
신호 발생
  ↓
리스크 평가 (사용자 설정 기반)
  ↓
거래 결정 (자동)
  ↓
주문 실행 (자동)
```

## 🛡️ 안전장치 (자동)

### 자동 중단 조건

1. 일일 손실 한도 도달 → **자동 중단**
2. 건강도 점수 < 30% → **자동 중단**
3. 연속 손실 5회 → **자동 중단**
4. API 연결 끊김 → **자동 중단**

### 자동 복구

- API 재연결 → **자동 재시도**
- 경미한 오류 → **자동 복구**

## 🎛️ 옵션별 동작

### 리스크 레벨

**낮음:**
- 포지션 크기: 기본값 × 0.5
- 거래 빈도: 낮음
- 손실 한도: 기본값 × 0.7

**보통 (기본값):**
- 포지션 크기: 기본값
- 거래 빈도: 보통
- 손실 한도: 기본값

**높음:**
- 포지션 크기: 기본값 × 1.5
- 거래 빈도: 높음
- 손실 한도: 기본값 × 1.3

### 거래 빈도

**낮음:**
- 신호 확인: 30초마다
- 최소 신호 강도: 높음 (0.7 이상)

**보통 (기본값):**
- 신호 확인: 5초마다
- 최소 신호 강도: 보통 (0.6 이상)

**높음:**
- 신호 확인: 1초마다
- 최소 신호 강도: 낮음 (0.5 이상)

## 💡 핵심 원칙

1. **최소한의 설정만**
   - 사용자가 정말 필요한 것만
   - 나머지는 모두 자동

2. **명확한 상태 표시**
   - 지금 뭐 하고 있는지 명확히
   - 문제가 있으면 알림

3. **안전 우선**
   - 위험하면 자동 중단
   - 복구 가능하면 자동 복구

4. **간단한 UI**
   - 복잡한 것은 숨기기
   - 필요할 때만 보이기
